# Contributing to AMTCE

First off, thank you for considering contributing to AMTCE! It's people like you that make the open source community such a great place to learn, inspire, and create.

## Getting Started

1. Fork the repository on GitHub.
2. Clone your fork locally.
3. Set up the development environment by installing `requirements.txt` in a virtual environment.
4. Ensure you have your `.env` configured properly (do NOT commit this file).

## Development Workflow

1. Create a branch for your feature or bug fix: `git checkout -b my-new-feature`
2. Make your changes and test them thoroughly.
3. Commit your changes with clear, descriptive commit messages.
4. Push to your fork and submit a Pull Request to the `main` branch.

## Code Standards

- Keep things simple! This project aims to be understandable to people who might not be deep into AI engineering.
- Comment extensively on 'magic numbers' (like specific coordinate crops, audio frequencies, or OpenCV thresholds).

## Need Help?

If you're stuck, please open a Draft PR or an Issue and ask for help. We are happy to guide you!
